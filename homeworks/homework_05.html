Exercise¶

Load the national COVID dataset and solve the exercise using scikit-learn library. Consider first the column corresponding to the (cumulative) number of deaths.

  1. Perform a train-test split, with observations from the last week corresponding to the test set;
  2. Build a suitable combination of kernels choosing from the ones shown in notebook_05;
  3. Fit a GaussianProcessRegressor to predict the daily number of swabs, plot future predictions and compare them to real test data;
  4. Repeat points 1-3 on the daily number of swabs, which can be computed from the cumulative number provided in the dataset.

You are welcome to try out and fit other data streams in the dataset.

from sklearn.gaussian_process.kernels import RBF, ExpSineSquared, RationalQuadratic, WhiteKernel, DotProduct
from sklearn.gaussian_process import GaussianProcessRegressor


def plot_predictions(gp, train_X,train_Y, test_X, test_Y, include_observed=True):

    x = np.hstack((train_X,test_X))
    X = x.reshape(-1,1)
    test_size = train_Y.shape[0]
    
    pred_y, pred_std = gp.predict(X, return_std=True)
    pred_y, pred_std = pred_y.flatten(), pred_std.flatten()
    plt.figure(figsize=(10, 7))
    if include_observed:
        plt.plot(x, list(train_Y) + list(test_Y), 'ok', alpha=0.1)
    l, = plt.plot(x, pred_y)
    plt.fill_between(x,
                     pred_y + pred_std,
                     pred_y - pred_std,
                     color=l.get_color(), alpha=0.3)
    plt.axvline(test_X[0], linestyle='--', color='k')
    plt.title(str(gp.kernel_));
    plt.show()
    pred_y, pred_std = gp.predict(test_X.reshape(-1,1), return_std=True)
    pred_y, pred_std = pred_y.flatten()[-test_size :], pred_std.flatten()[-test_size :]
    err = np.sum(((pred_y - test_Y) / pred_std) ** 2) / test_size 
    print("Log Marginal Likelihood:" + str(gp.log_marginal_likelihood_value_))
    print("Average distance test-prediction in units of sigma: " + str(err))
    return
    
    
def fit_and_predict(ker, train_X,train_Y, test_X, test_Y, seed = None):
    gp = GaussianProcessRegressor(kernel=ker, alpha=0.01,
                               normalize_y=False,
                               n_restarts_optimizer=10, random_state=seed)
    gp.fit(train_X.reshape(-1,1),train_Y.reshape(-1,1))
    plot_predictions(gp, train_X,train_Y, test_X, test_Y)
    return gp

def cumulative_to_daily(data):
    daily = [data[i + 1]- data[i] for i in range(len(data) - 1)]
    daily.insert(0, data[0])
    return np.array(daily)

The log marginal Likelihood is far better without the periodic term, but the estimated test error is smaller. This means that a more convoluted kernel better represents our data, but is far more difficult to optimize.

The cost of introducing the periodic term is that we are imposing a fixed period of the oscillations, due to weekends. This assumption fails because the oscillations caused by the weekend are negligible w.r.t. ones caused by "note_it" variable: a lot of data is affected by a bias.

Since a lot of times regions aggregate unreported cases in a single day, the oscillations are probably caused mainly by this aggregation of data in time
